"For the People" is an Indian news synthesis app that shows you the same news story
through multiple political lenses simultaneously — right, left, and centre — without
telling you what to think.

The core insight: Indian media is deeply politically polarized. OpIndia and The Wire
cover the same event as if they happened on different planets. Rather than pretending
neutrality (which is itself a political position), this app:

  1. Ingests articles from 35+ sources across the Indian political spectrum
  2. Clusters articles about the same real-world event
  3. Runs a synthesis through Gemini 2.0 Flash (or Groq fallback) that produces a
     structured JSON with:
     - Established facts (adjudicated by courts, official records)
     - Reported facts (3+ sources, 2+ lean buckets within 24h)
     - Contested claims: right narrative, left narrative, other narrative
     - Named individuals with procedural status (convicted/charged/alleged)
     - Rhetoric flags (red herrings, broken promises, evasion, factual falsehoods)
     - Statistics with source citations
     - Common ground proposals or irreconcilable disagreements
  4. Presents all of this in a newspaper-aesthetic mobile UI

The app is NOT a neutral aggregator. It actively applies epistemological rules to
distinguish facts from narratives. It names names. It flags politicians' lies equally
regardless of party.

Target user: Indians who are tired of consuming only one side's reality bubble.

  id              text PK

AUTO PIPELINE (NEW — primary data flow)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

File: apps/web/src/app/api/cron/pipeline/route.ts
Trigger: Vercel Cron, fires every minute (vercel.json: "schedule": "* * * * *")
Auth: Vercel auto-injects CRON_SECRET; route checks Authorization: Bearer {CRON_SECRET}
      (skipped locally when CRON_SECRET env var is not set)
maxDuration: 300 seconds (5 min), with internal 4.5-min safety cutoff

Each tick does:
  1. INGEST (parallel):
     - 12 RSS feeds fetched concurrently using built-in XML parser (no rss-parser dep needed)
       NDTV, The Hindu, Times of India, Indian Express, Hindustan Times, India Today,
       The Wire, Scroll.in, Republic World, News18, The Print, Firstpost
     - 7 Brave News API queries (India politics/crime/economy/sports/tech/science/foreign)
     - Deduplicates by URL (ON CONFLICT DO NOTHING)
     - Extracts: title, published_at, body_excerpt, image_url

  2. CLUSTER:
     - Loads all articles without cluster_id
     - Jaccard similarity on title tokens (threshold 0.22, 72h time window)
     - Requires ≥2 articles from ≥2 different sources to form a cluster
     - Calls Gemini to generate a neutral canonical title
     - Inserts cluster row, updates article.cluster_id

  3. SYNTHESIZE (max 3 per run, respects time budget):
     - Loads pending clusters (status='pending'), processes up to 3
     - Calls Gemini 2.0 Flash with the full synthesis system prompt
     - Parses + validates output against SynthesisOutputSchema (Zod)
     - Inserts into syntheses table, marks cluster status='synthesized'

Response: { ok, elapsed, log[], timestamp }

NOTE: Vercel Hobby plan only allows daily crons. Every-minute crons require Vercel Pro.
      If on Hobby plan, change schedule to "0 * * * *" (hourly) or upgrade.
      Required env vars in Vercel dashboard: GEMINI_API_KEY, BRAVE_API_KEY, DATABASE_URL

MANUAL WORKER PIPELINE (fallback / backlog processing)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Run from tristhana/apps/worker/:

  pnpm --filter worker ingest:once
    → Fetches all 35+ RSS feeds + NewsAPI.org (if NEWS_API_KEY set) + Guardian API
    → For each new article: extracts body text, image, stores in articles table

  pnpm --filter worker cluster:once
    → Takes unclustered articles
    → Groups by Jaccard keyword similarity + 72h time window
    → Creates/updates cluster rows; calls Groq LLaMA-3.1-8b for canonical titles
    → Links articles to clusters

  pnpm --filter worker synth:one [cluster_id]
    → Takes ONE 'pending' cluster (or specific cluster_id if provided)
    → Builds synthesis prompt with all cluster articles
    → Calls Groq LLaMA-3.3-70b-versatile
    → Parses JSON response against SynthesisOutput Zod schema
    → Stores in syntheses table, marks cluster as 'synthesized'
    → Groq free tier: 100k tokens/day — main bottleneck for backlog

  pnpm --filter worker link:once
    → Scans synthesized clusters for shared named entities
    → Creates cluster_links rows for story graph navigation

Typical workflow for fresh data (manual):
  1. pnpm --filter worker ingest:once
  2. pnpm --filter worker cluster:once
  3. Repeat: pnpm --filter worker synth:one (until quota hit or all done)
  4. pnpm --filter worker link:once

One-off scripts (run from repo root with node):
  node scripts/scrape-wb-election.mjs     ← Scrapes a topic via Brave, creates cluster

  node scripts/insert-synth.mjs           ← Batch-inserts 16 pre-written syntheses

SYNTHESIS PROMPT (The core epistemology)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Located at: packages/shared/src/synthesis-prompt.ts
Used by: apps/worker/src/synth.ts (Groq) AND apps/web/src/app/api/cron/pipeline/route.ts (Gemini)

The prompt enforces three tiers of factual confidence:
  Tier 1: Adjudicated facts (courts, official commissions, RTI, autopsy reports)
  Tier 2: Reported facts (3+ outlets, 2+ lean buckets, within 24h)
  Tier 3: Contested claims (motive, blame, sequence — diverge by lean)

Key rules:
  - Name individuals; never anonymize when sources have named them
  - Apply identical language for identical procedural status regardless of party/religion
  - "Convicted of rape" not "convicted in a sexual offense matter"
  - Don't construct causal chains — extract which prior events each SIDE'S outlets invoked
  - Flag RED_HERRING, BROKEN_PROMISE, EVASION, THREAT, FACTUAL_FALSEHOOD for all parties
  - Never invent statistics; empty array beats fabricated entry
  - Counterweight LLM liberal bias when constructing right_narrative (steelman from sources)
  - model_uncertainty_notes is required, never empty

Output JSON schema (simplified):
  {
    story_age_band: "breaking"|"developing"|"mature"|"historical",
    established_facts: [{claim, tier: 1, citations}],
    reported_facts: [{claim, tier: 2, citations}],
    contested_claims: {
      right_narrative: {summary (4-6 sentences), key_claims, framing_devices, sources_used},
      left_narrative: {same},
      other_narrative: {same} | null
    },
    named_individuals: [{name, role, party, procedural_status, specific_acts, status_evidence}],
    precedents_cited_by_right/left/other: [{event, year, citing_outlets}],
    rhetoric_flags: [{type, leader_name, party, quoted_statement, context, contradicting_fact}],
    statistics: [{claim, value, unit, year, source_authority, self_confidence}],
    common_ground: [{proposal, why_right_accept, why_left_accept, why_fail}] | null,
    irreconcilable_disagreements: [string],
    model_uncertainty_notes: string
  }

  politics      → /api/feed/category?cat=politics

Feed sorting (ALL channels):
  ORDER BY (SELECT MAX(published_at) FROM articles WHERE cluster_id = clusters.id) DESC NULLS LAST
  Sorts by the most recent article in each cluster, not by cluster insert time.

Positive feed filtering (2 layers):
  1. NEGATIVE_WORDS blocklist: death, deaths, poison, attack, kills, violence, crash, etc.
     A story matching any negative word is EXCLUDED even if synthesis has common_ground.
  2. POSITIVE_WORDS allowlist: award, achievement, success, celebration, etc.
     OR synthesis.output.common_ground is non-empty array.

Infinite scroll:
  - page.tsx loads 20 clusters server-side (force-dynamic, not cached)
  - Client fetches /api/feed?offset=N for more
  - Total count via raw SQL: SELECT cast(count(*) as int) FROM clusters
    (NOT db.$count() — unreliable due to Drizzle duplicate package conflict)

ARCHITECTURE DECISIONS & WHY
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

WHY NEON POSTGRESQL (not SQLite):
  SQLite worked locally but Vercel serverless functions can't persist a SQLite file.
  Neon provides serverless PostgreSQL with HTTP-based connection (no TCP pool needed).
  @neondatabase/serverless driver works in Vercel Edge/Node environments.

WHY DRIZZLE ORM:
  Type-safe, Neon-compatible, minimal bundle size. Zod integration for schema validation.

WHY GEMINI FOR CRON SYNTHESIS (not Groq):
  Gemini 2.0 Flash is free with generous rate limits on the provided API key.
  The cron pipeline needs to synthesize continuously; Groq's 100k/day free tier runs out.
  Gemini also supports systemInstruction field natively, matching the prompt structure.

WHY GROQ FOR TRAIL BUILDER (not Claude/Gemini):
  The trail builder makes many tool calls (Brave search) in a single agentic session.
  Groq's LLaMA-3.3-70b is fast and its API has good function-calling support.
  Claude is more expensive; Gemini trail builder not yet implemented.

WHY BRAVE NEWS API FOR CRON INGEST:
  Free plan, Indian results good, freshness=pd gives past-24h articles.
  No extra dependency needed (pure fetch). Works from Vercel serverless.

WHY CUSTOM XML PARSER IN CRON (not rss-parser):
  Avoids adding rss-parser as a dependency to apps/web.
  The hand-rolled regex XML parser handles the specific RSS patterns from Indian news sites.
  If it misses edge cases, rss-parser can be added: npm install rss-parser -w apps/web

WHY CORRELATED SUBQUERY FOR FEED SORT:
  SELECT MAX(published_at) FROM articles WHERE cluster_id = clusters.id
  This sorts clusters by their freshest article, not by cluster insert time.
  Avoids JOIN + GROUP BY complexity. Slightly slower than a JOIN but acceptable at this scale.

WHY TRANSPARENT SWIPE OVERLAY ON VIDEO:
  YouTube iframe creates a cross-origin boundary — touch events inside iframe don't bubble
  to parent DOM elements. Transparent div at z-index:5 positioned over the iframe captures
  swipe gestures. Custom mute button at z-index:10 sits above the overlay.
  Native YouTube controls (volume, pause, fullscreen) inside the iframe are unreachable;
  mute is handled via postMessage with enablejsapi=1 parameter.

WHY createPortal FOR TRAIL DRAWER:
  Story cards use CSS transform for video slide animation.
  CSS transform creates a new containing block, so position:fixed children render relative
  to the transformed element, not the viewport. createPortal escapes this hierarchy.

WHY INLINE STYLES (not Tailwind/CSS modules):
  Rapid iteration without needing class names. CSS variables handle theming.

WHY HORIZONTAL SWIPE FOR CATEGORIES (not tabs):
  Mobile-first. Swiping between categories feels natural. The 9-panel CSS transform
  approach keeps all panels rendered (so no unmount/remount on channel switch),
  which means expanded state persists across swipes.

CRITICAL TECHNICAL GOTCHAS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. CSS TRANSFORM CONTAINING BLOCK
   If you wrap ANY fixed-position UI (modals, drawers, tooltips) inside a container that
   has transform, filter, or will-change CSS, it will NOT be positioned relative to viewport.
   Solution: always use createPortal to mount such elements directly in document.body.

2. INTERSECTION OBSERVER + CSS TRANSFORM SCROLL SENTINELS
   The 9-panel swipe layout uses CSS translateX on the outer container. Panels at different
   horizontal positions only have their scroll sentinel intersect with viewport when visible.
   One IntersectionObserver per panel works correctly because all panels are in DOM simultaneously.

3. db.$count() IS UNRELIABLE (DRIZZLE DUPLICATE PACKAGE BUG)
   Drizzle's $count() returns undefined when two drizzle-orm versions are installed.
   Always use: db.select({ count: sql<number>`cast(count(*) as int)` }).from(table)
   Then access: rows[0]?.count ?? 0

4. FORCE-DYNAMIC REQUIRED FOR FEED ROUTES
   Without export const dynamic = 'force-dynamic', Next.js caches the server component
   output and serves stale HTML. All feed API routes and page.tsx use force-dynamic.

5. YOUTUBE AUTOPLAY REQUIRES mute=1
   Mobile browsers (iOS Safari, Chrome Android) block autoplay with sound.
   Add mute=1&playsinline=1 to embed URL. Use postMessage API (enablejsapi=1) for custom mute toggle.
   Without mute=1, the iframe shows a static thumbnail with the red YouTube play button.

6. HYDRATION MISMATCH WITH INLINE THEME SCRIPT
   layout.tsx has an inline script that reads localStorage and sets data-theme on <html>
   before React hydrates. Fix: suppressHydrationWarning on the <html> element.

7. overflow-anchor: none ON BODY
   Without this, when a story card expands, browser scroll anchoring causes page to jump.
   Disable with overflow-anchor: none on body in globals.css.

8. PNPM WORKSPACES DRIZZLE DUPLICATE
   drizzle-orm is installed in both apps/web/node_modules and packages/db/node_modules.
   TypeScript sees two different class definitions. DOES NOT affect runtime. Safe to ignore.

9. WINDOWS ESM DYNAMIC IMPORT PATH
   On Windows, import(absolutePath) requires a file:// URL, not a raw path.
   Use pathToFileURL() from 'url' module: await import(pathToFileURL(absolutePath).href)
   (This was the fix for scripts/insert-wb-synth.mjs and scrape-wb-election.mjs)

  - Worker pipeline (apps/worker): runs locally or on any Node.js host,
    uses Groq for synthesis, no Vercel plan restrictions.
  They can run simultaneously without conflict (DB upserts are idempotent).
